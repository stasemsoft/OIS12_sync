0.0.1. OIS12-specifieke informatie

Wen je zo snel mogelijk een professionele houding aan. Inhoudelijk wil dat zeggen dat je de dingen die je al geleerd hebt toepast, probeert onderhoudbare en leesbare code te schrijven, nadenkt over een probleem en de oplossing en niet direct je IDE opstart en begint te typen.

Als je ergens een oplossing of een stuk code vindt dat je kunt gebruiken, vermeld dit dan! Bijvoorbeeld door de url waar je het gevonden hebt in comment in je code te copy pasten. Behalve dat dit algemeen goed gebruik is (reproduceerbaarheid) maakt het onderhoud van je programma gemakkelijker!

Vraag regelmatig feedback op je code aan ervaren ontwikkelaars, aan een softwaredocent, aan een teamgenoot, aan iemand anders die net zover is als je zelf bent. Leesbare code schrijven is namelijk bijna onmogelijk als je niet weet hoe anderen naar code kijken.

Dat je bereid bent moeite te doen om uitleg te geven aan een collega (klasgenoot?) die niet verder komt met een probleem. Ook is het goed te oefenen zelf een vraag te stellen aan een collega.

Als ICTer werk je in een zich steeds sneller ontwikkelend vakgebied: hoe slim je ook bent en hoe goed je ook kunt programmeren, je zult continu bijven leren en het is onmogelijk alles te weten, je zult je collega's keihard nodig hebben!

0.0.2. Wat wil jij programmeren?

OIS12 bereidt je voor op semester 2 van ICT & Technology en ICT & Software Engineering. Het doel is het opbouwen van vaardigheid voor het ontwikkelen je bruikbare, onderhoudbare programma's schrijven in C#.

0.0.3. Leeractiviteiten

Je krijgt periodiek feedback van je docent die wordt genoteerd in Canvas. Bekijk zelf aan de hand van deze feedback welke programmeer-oefeningen je gaat maken. Bij OIS12 ga je zo veel mogelijk zelf aan de slag met programmeren. Door veel te oefenen krijg je het vak snel onder de knie.
